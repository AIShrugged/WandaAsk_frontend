# Report — 26.03.2026

## Summary

- ♻️ Artifact types and UI components moved to a dedicated `entities/artifact`
  layer
- 🏗️ Removed the old static `features/analysis` feature (replaced by
  artifact-based rendering)

---

### ✨ New features

- **Live analysis artifacts on follow-ups and meetings**: The "Analysis" tab on
  both the follow-up detail page and the meeting detail page now loads visual
  artifacts produced by the AI in the chat, instead of a static text block. The
  page fetches whichever chat is linked to the , pre-loads its artifacts on the
  server, and then refreshes them every 5 seconds on the client — so new results
  appear without a page reload. A manual refresh button is also available.

### 🔧 Improvements

- **Empty states for analysis tab**: When a methodology has no linked chat yet,
  or the chat has no artifacts, a clear empty state message is shown explaining
  what to do next (open the AI chat and describe the evaluation criteria).

### ♻️ Refactoring

- **`entities/artifact` — canonical home for artifact code**: All artifact types
  (`TaskTable`, `MeetingCard`, `PeopleList`, `InsightCard`, `Chart`,
  `TranscriptView`, `MethodologyCriteria`) and their renderers were moved from
  `features/chat` into a new `entities/artifact` layer. The chat feature now
  imports from `entities/artifact`, keeping FSD boundaries clean.
  `features/chat/model/types.ts` still re-exports the types for backward
  compatibility.

- **Removed `features/analysis`**: The old static analysis feature (donut
  charts, linear progress bars, conclusion items, min/max metrics) was deleted
  entirely. It has been replaced by the artifact-based approach described above.
  All related tests were also removed.

### 🐛 Bug fixes

- Fixed multiple issues with dynamic methodology generation (three separate fix
  commits over the course of the day — covers generation logic and code style
  corrections).

### 🏗️ Infrastructure

- Large-scale lint/format cleanup across nearly all files in the project
  (trailing blank lines, import ordering, minor style issues). No behavior
  changes.
