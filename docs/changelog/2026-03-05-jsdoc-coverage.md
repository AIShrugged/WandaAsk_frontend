## Подход

Написан Node.js скрипт-автоматизатор в три прохода:

### Проход 1 — основной (291 файл)

Обработаны стандартные паттерны:

- `export function Foo(a, b)` / `export async function Foo(...)`
- `export const Foo = (...) =>`
- `class Foo`

Для каждой функции автоматически сгенерирован JSDoc с учётом:

- **React-компоненты** (имя с заглавной буквы, `.tsx`):
  `/** FooComponent component. */`
- **Хуки** (начинаются с `use`): `/** useBar hook. */`
- **Утилиты и Server Actions**:
  `/** funcName. @param x - x. @returns Promise. */`
- Деструктурированные параметры `({ a, b })` → `@param props - Component props.`

### Проход 2 — edge cases (23 файла)

Дополнительно обработаны:

- Конструкторы классов: `constructor(message, options) {`
- Методы внутри объектных литералов: `hydrate: (items, count) => {`
- Тест-файлы (`__tests__`): добавлена директива `eslint-disable` в начало файла

### Проход 3 — многострочные сигнатуры (76 файлов)

Функции с параметрами, разбитыми на несколько строк:

```ts
// До
export async function getChats(
  offset = 0,
  limit = 20,
): Promise<...>

// После
/**
 * getChats.
 * @returns Promise.
 */
export async function getChats(
  offset = 0,
  limit = 20,
): Promise<...>
```

Также дженерики: `createCachedListStore<T>()` — `<T>` больше не ломает парсинг.

## Проблемы и решения

### 1. `execSync` падает при ненулевом exit code ESLint

ESLint возвращает код 1 при наличии предупреждений. `execSync` выбрасывает
исключение.

**Решение:** заменён на `spawnSync`, который не кидает исключений и позволяет
читать `stdout` независимо от кода выхода.

### 2. Многострочные сигнатуры функций

Regex на одну строку не матчил:

```ts
export async function getChats(
  offset = 0,     // ← ESLint указывал на эту строку
```

**Решение:** в третьем проходе используется упрощённый детектор по первой строке
— извлекается имя и флаг `async` без попытки парсить параметры.

### 3. Дженерики ломали regex

`createCachedListStore<T>()` → `<T>` прерывал `\w+` матч.

**Решение:** regex для извлечения имени остановлен на `\w+` без захвата типовых
параметров.
