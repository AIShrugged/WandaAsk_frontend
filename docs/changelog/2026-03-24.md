### 🆕 Новая функциональность

#### Kanban-доска (`features/kanban/`, `app/dashboard/kanban/`)

Полная реализация Kanban для задач (issues):

- **`features/kanban/model/types.ts`** — типы `KanbanCard`, `KanbanFilters`,
  `IssueStatus`, константы `KANBAN_COLUMNS`, `KANBAN_PRIORITY_LABELS`
- **`features/kanban/api/kanban.ts`** — `getKanbanIssues(filters)` (SSR),
  `moveKanbanCard(id, status, card)` (Server Action с optimistic update)
- **`features/kanban/ui/kanban-board.tsx`** — клиентская доска:
  - 4 колонки: Open / In Progress / Paused / Done
  - Drag-and-drop между колонками с optimistic update + rollback при ошибке
  - Hover-превью карточки в боковой панели (`TaskPreviewPanel`): описание,
    приоритет, story points, ассайни, дедлайн, вложения/комментарии
  - Контекстное меню «Move to» для каждой карточки
  - Фильтры: организация, команда, поиск (client-side)
  - Синхронизация колонок при изменении URL-параметров (server re-render)
- **`app/dashboard/kanban/page.tsx`** — SSR страница, передаёт initial data в
  `KanbanBoard`
- **`features/menu/lib/options.ts`** — добавлен пункт меню «Kanban»
  (`SquareKanban`); Statistics переключён на иконку `BarChart2`
- **`shared/lib/routes.ts`** — добавлен `ROUTES.DASHBOARD.KANBAN`

### 🐛 Исправления

**401-редирект в httpClient** (`shared/lib/httpClient.ts`):

- Было: `redirect('/api/auth/clear-session')` — несуществующий маршрут
- Стало: `redirect(ROUTES.AUTH.LOGIN)` → `/auth/login`

**Страница meeting без followUp** (`app/dashboard/meeting/[id]/page.tsx`):

- Добавлена проверка `if (!followUp)` — рендерит «No summary available for this
  meeting yet.» вместо краша

**Кэш динамических страниц** (`next.config.ts`):

- `staleTimes.dynamic`: 30 → 0 — отключён кэш, страницы всегда рендерятся
  свежими

**Error boundary** (`app/dashboard/error.tsx`):

- Добавлен `min-h-0 overflow-y-auto items-start` — длинные ошибки теперь
  скроллятся
