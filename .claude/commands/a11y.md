## Checklist

### Semantic HTML

- Proper heading hierarchy (h1 > h2 > h3, one h1 per page)
- Use semantic elements: `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`,
  `<footer>`
- Lists use `<ul>`/`<ol>` + `<li>`
- Buttons are `<button>`, links are `<a>`, not `<div onClick>`

### Forms (critical for this project)

- All inputs have associated `<label>` elements
- Error messages linked via `aria-describedby` (as in `login-form.tsx` pattern)
- Error announcements use `role="alert"` and `aria-live="polite"`
- Required fields marked with `aria-required`
- Form groups use `<fieldset>` + `<legend>`

### Keyboard navigation

- All interactive elements focusable via Tab
- Focus order matches visual order
- Focus indicators visible (not `outline: none` without replacement)
- Modals trap focus, Escape closes them
- Custom dropdowns keyboard-accessible (Arrow keys, Enter, Escape)

### ARIA

- Interactive widgets have appropriate ARIA roles
- Loading states communicated with `aria-busy`
- Dynamic content uses `aria-live` regions
- Icons have `aria-hidden="true"` or meaningful `aria-label`

### Color & contrast

- Text meets WCAG AA contrast ratio (4.5:1 normal, 3:1 large)
- Information not conveyed by color alone
- Focus indicators have sufficient contrast

Report issues with file paths, line numbers, and WCAG criteria references.
