### ✨ Новые функции

- **Удаление демо-данных**: пользователи теперь могут удалить ранее
  сгенерированные демо-данные прямо из выпадающего меню демо-кнопки — появляется
  кнопка "Delete demo data" с иконкой корзины, спиннером во время удаления и
  уведомлением об успехе. Новый Server Action `deleteDemo()` вызывает
  `DELETE /demo` на бэкенде и перезагружает страницу после успеха.

- **Экспорт follow-up отчёта**: на странице анализа follow-up добавлена кнопка
  экспорта с выбором формата (PDF / Excel / HTML). Новый Route Handler
  `GET /api/follow-ups/[id]/export` проксирует бинарный файл с Laravel-бэкенда
  напрямую в браузер, используя httpOnly-cookie для авторизации.

- **Вкладка задач на странице встречи**: на странице встречи добавлена вкладка
  "Tasks" с компонентом `MeetingTasks`, который показывает список задач,
  извлечённых ИИ из транскрипта: заголовок, описание, исполнитель, срок и
  цветные статус-бейджи (Open / In progress / Done / Cancelled).

- **Управление внешними идентификаторами в профиле**: в разделе профиля
  пользователя появилась секция "Linked Identities" для подключения и отключения
  внешних аккаунтов (Telegram, Zoom и др.). Реализовано с оптимистичными
  обновлениями через `useOptimistic` + `useTransition` без перезагрузки
  страницы. Новые Server Actions: `getIdentities`, `linkIdentity`,
  `unlinkIdentity`.

- **Флаг `is_demo` в профиле пользователя**: сущность `UserProps` дополнена
  полем `is_demo: boolean`, позволяющим компонентам реагировать на наличие
  демо-данных у пользователя.

### 🔧 Улучшения

- **Google Calendar OAuth**: исправлен поток подключения — ошибка
  `SOURCE_ALREADY_EXISTS` теперь обрабатывается отдельно с понятным сообщением
  "Google Calendar is already connected to your account." Добавлено логирование
  ошибок API. Разделён JSON.parse и throw для устранения хрупкого сравнения
  строк в коде ошибки.

- **Демо-кнопка**: статус демо-данных теперь отслеживается при монтировании
  компонента — если данные уже существуют (`hasExistingDemo`), кнопка "Delete
  demo data" показывается сразу без лишних действий пользователя.

- **`getSources()`**: переписан, возвращает строго типизированный `Source[]`
  вместо `unknown`.

- **`CalendarAttachedToast`**: убирает параметр `?attached=1` из URL после
  показа тоста, чтобы уведомление не появлялось повторно при обновлении
  страницы.

### 🐛 Исправления

- **Удалён обход TLS-проверки**: убран `NODE_TLS_REJECT_UNAUTHORIZED=0` из
  `next.config.ts` — бэкенд теперь использует валидный сертификат.

- **`fetchDebugger`**: добавлено исключение для `telemetry.nextjs.org`, чтобы
  избежать ошибки несовместимости с `AbortSignal`.

- **Страница встречи**: исправлен относительный импорт на `@/`-алиас.

- **`middleware.ts` переименован в `proxy.ts`**: следует соглашению Next.js 16,
  где `middleware.ts` — зарезервированное имя.

- **Проверка `due_date` в задачах**: добавлена защита `Number.isNaN` перед
  форматированием даты, чтобы избежать падения при невалидном значении.

- **`identities.ts`**: небезопасное приведение `json.data` заменено на
  null-guard.
