# 2026-03-04 — Генерация экрана авторизации в Figma

## Что делали

Воспроизвели текущий экран авторизации приложения Tribes в Figma на основе
существующего кода фронтенда.

Исходники, по которым строился макет:

- `app/auth/login/page.tsx` — структура страницы
- `app/auth/layout.tsx` — фоновый лейаут (muted bg, dot-pattern)
- `features/auth/ui/login-form.tsx` — форма входа
- `features/auth/ui/auth-form-footer.tsx` — кнопка и ссылка
- `features/auth/lib/fields.tsx` — конфигурация полей
- `shared/ui/brand/TribesLogo.tsx` — логотип
- `shared/ui/card/Card.tsx` — карточка
- `app/globals.css` — все цветовые токены и радиусы

Итоговый макет включает:

- Фон `#f2f2f4` (muted), 1440×900 px
- Логотип Tribes — зелёный бокс с «T» + текст
- Карточка (400 px) с тенью и скруглением 8 px
- Заголовок «Sign in» + подзаголовок
- Поля Email и Password с лейблами и плейсхолдерами
- Зелёная кнопка «Sign in» (#4FB268)
- Ссылка «Don't have an account? Register»

Макет размещён на странице **Auth Screen** в файле Figma.

## Проблемы и решения

### 1. Подключение к Figma Desktop

Ни CDP (`--remote-debugging-port=9222`), ни плагин Desktop Bridge изначально не
были настроены.

**Решение:** установили Desktop Bridge plugin через ручной импорт
`manifest.json` из npm-кэша (`~/.npm/_npx/.../figma-desktop-bridge/`).

### 2. Неверное название шрифта

Попытка загрузить `Inter SemiBold` вызвала ошибку — в Figma стиль называется
`Semi Bold` (с пробелом).

**Решение:** предварительный запрос `figma.listAvailableFontsAsync()` для
получения точных названий.

### 3. Устаревший API смены страницы

`figma.currentPage = page` вызвало ошибку:
`Cannot call with documentAccess: dynamic-page`.

**Решение:** использован `await figma.setCurrentPageAsync(page)`.

### 4. `FILL` sizing до добавления в родителя

`layoutSizingHorizontal = 'FILL'` выбрасывал ошибку при установке до добавления
ноды в auto-layout родителя.

**Решение:** отказались от FILL в пользу явных фиксированных ширин
(`INNER_W = CARD_W - padding * 2 = 336 px`). Более предсказуемый подход для
сложных вложенных лейаутов.

## Вывод

Макет экрана авторизации успешно воспроизведён в Figma с точным соответствием
цветовым токенам, отступам и типографике из кода.

**Сильные стороны:**

- Полная синхронизация дизайна с кодом: все цвета, радиусы и отступы взяты
  напрямую из `globals.css` — нет расхождений между макетом и продакшеном
- Высокая скорость итерации: 6 версий кода за одну сессию, каждая на основе
  конкретной диагностированной ошибки
- Воспроизводимый подход: скрипт можно запустить повторно и получить актуальный
  макет при изменениях в коде
- Успешно настроена интеграция Figma MCP для дальнейшего использования в проекте
- Объём написанного кода (~888 строк) полностью покрывает все компоненты экрана
  с диагностическими итерациями

**Слабые стороны:**

- Потребовалось 6 итераций из-за ограничений Figma Plugin API (FILL-sizing,
  шрифты, page API)
- Dot-pattern фон из CSS (`radial-gradient`) не воспроизводится нативно в Figma
  — пропущен как визуальная деталь
